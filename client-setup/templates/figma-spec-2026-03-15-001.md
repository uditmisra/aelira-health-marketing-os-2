## 2. Layer Naming Convention — Non-Negotiable

The Google Sheets data plugin reads these exact names to auto-populate your ad frames. A single typo breaks the automation. Use copy-paste.

| Layer name | What it is | What the plugin does |
|---|---|---|
| `headline` | Main headline text layer | Replaces with each ad's headline |
| `body_copy` | Body copy text layer | Replaces with body copy |
| `cta` | Text inside the button | Replaces with CTA text |
| `logo` | Logo image layer | Toggles visibility only — plugin won't change it |
| `background` | Background rectangle or image | Can swap the image fill |
| `cta_button` | Group: button shape + `cta` text layer | Contains `cta` layer inside |

**Beginner tip:** In Figma's Layers panel (left sidebar), double-click a layer name to rename it. Names are **case-sensitive** — `headline` ≠ `Headline`. After renaming, click elsewhere to confirm. Use Cmd+C / Cmd+V to paste layer names from this doc.

### Frame 1 — LinkedIn Single Image

```
FRAME NAME (in Layers panel): LinkedIn Single Image
Canvas size: 1200 × 627 px
Background: #F0EBE0 (warm cream) — or a clinical photograph

Build layers in this order (top = frontmost in Layers panel):

────────────────────────────────────
1.  headline
    Type:       Text layer
    Font:       DM Sans, 700 (Bold)
    Size:       52–60px
    Color:      #1B5E30
    Max chars:  60  ← LinkedIn hard limit for readability
    Position:   Left edge: 64px from left
                Vertical:  centered in upper 55% of frame
    Width:      680px (Fixed Width — set in text properties)
    Line height: 1.15

    Beginner tip: Click T (text tool) → click and drag to draw
    a text box 680px wide. In the right panel, set Constraints
    to Fixed Width. This makes text wrap instead of overflow.

────────────────────────────────────
2.  body_copy
    Type:       Text layer
    Font:       DM Sans, 400 (Regular)
    Size:       22px
    Color:      #6B7B6E
    Max chars:  100
    Position:   Same left edge (64px), 20px below bottom of headline
    Width:      600px (Fixed Width)
    Line height: 1.5

────────────────────────────────────
3.  cta_button  ← This is a GROUP containing items 3a and 3b
    Position:   64px from left, 28px below body_copy

    3a. Rectangle (button shape)
        Size:         220 × 60px
        Fill:         #2E6B40
        Corner radius: 50px (pill)

    3b. cta  ← Text layer INSIDE the group
        Font:       DM Sans, 600 (SemiBold)
        Size:       20px
        Color:      #FFFFFF
        Max chars:  20
        Alignment:  Center (both horizontally and vertically within button)

    Beginner tip: Draw the rectangle first (R key). Then draw
    the text on top. Select both → right-click → Group Selection
    (Cmd+G). Rename the group to cta_button. Rename the text
    layer inside to cta.

────────────────────────────────────
4.  logo
    Type:       Image
    Source:     https://aelira.in/aelira-logo-v2.png
                (download it first: right-click → Save Image)
    Height:     40px, width auto (proportional)
    Position:   Top-left: 40px from left, 40px from top

    Beginner tip: File menu → Place image (or drag the PNG
    file onto the Figma canvas). Resize by holding Shift and
    dragging a corner handle.

────────────────────────────────────
5.  background  ← Must be the BOTTOM layer
    Type:       Rectangle
    Size:       1200 × 627px (full frame)
    Fill:       #F0EBE0
    Position:   0, 0

    Beginner tip: Press R → draw to fill frame. Then press
    Cmd+[ repeatedly to send it to the back.
```

**What the finished ad looks like:** Cream background. Dark green bold headline on the left. Gray-green body copy below. Pill-shaped green button. Logo top-left. Clean, calm, clinical.

### Frame 3 — LinkedIn Carousel (Master Slide)

```
FRAME NAME: Carousel — Master
Canvas size: 1080 × 1080 px

CARROUSEL RULES:
  Slide 1 (Intro):   Headline + CTA button. Grabs attention in feed.
  Slides 2–N:        One point per slide. Headline + optional body copy.
                     Hide cta_button (set opacity to 0).
  Final slide:       CTA button prominent. Dark green background for contrast.

────────────────────────────────────
1.  headline
    Font:       DM Sans, 700
    Size:       52–64px
    Color:      #1B5E30  ← Change to #FFFFFF on dark final slide
    Max chars:  80  (LinkedIn max is 255 per slide; 80 reads better)
    Position:   72px margins, vertically centered in frame
    Width:      936px (Fixed Width)
    Line height: 1.2

────────────────────────────────────
2.  body_copy
    Font:       DM Sans, 400
    Size:       24px
    Color:      #6B7B6E  ← Change to #F0EBE0 on dark final slide
    Max chars:  100
    Position:   72px margins, 24px below headline
    Width:      936px (Fixed Width)
    Note: On headline-only slides, keep the layer but set opacity to 0.
          This ensures the plugin doesn't error if body_copy is empty.

────────────────────────────────────
3.  slide_number  ← DECORATIVE, not automated
    Font:       DM Sans, 600
    Size:       18px
    Color:      #86D08A (mint)
    Content:    e.g. "01 / 05"
    Position:   Top-right, 72px from right edge, 48px from top
    Note:       Update manually for each slide. Plugin ignores this.

────────────────────────────────────
4.  cta_button (Group)
    4a. Rectangle: 280 × 72px, radius 50px, fill #2E6B40
        ← On dark final slide: change fill to #86D08A (mint)
    4b. cta text: DM Sans 600, 22px, #FFFFFF, max 20 chars
        ← On dark final slide: change color to #1B5E30
    Position: Bottom-left, 72px from left, 80px from bottom
    On content slides (not intro/final): set group opacity to 0

────────────────────────────────────
5.  logo
    Height: 40px, width auto
    Position: Top-left, 48px from left, 48px from top
    Color variant: Standard logo for light slides
                   White variant for dark final slide
                   (placeholder until white variant is available)

────────────────────────────────────
6.  background
    Light slides:  Fill #F0EBE0
    White slides:  Fill #FFFFFF
    Dark final slide: Fill #1B5E30
    Size: 1080 × 1080px, bottom of stack

────────────────────────────────────
HOW TO BUILD THE FULL CAROUSEL:

  Step 1: Build this master slide completely.
  Step 2: Duplicate it (right-click frame → Duplicate, or Cmd+D).
  Step 3: For each duplicate:
          - Update headline text
          - Toggle body_copy opacity (0 = hidden, 100 = visible)
          - Update slide_number manually
          - Toggle cta_button opacity (0 on middle slides, 100 on intro+final)
  Step 4: For the FINAL slide:
          - Set background fill to #1B5E30 (dark green)
          - Change headline color to #FFFFFF
          - Change body_copy color to #F0EBE0
          - CTA button: fill #86D08A, text color #1B5E30
  Step 5: Export all carousel frames:
          Select all → Export (bottom right panel) → PNG 1× → Export.
  Step 6: Upload to LinkedIn Ads as a document/carousel creative.
```

## 4. Auto-Population Setup Guide

This is how the Google Sheets → Figma pipeline works. Once set up, every ad copy generation run can be pushed to Figma in under 5 minutes.

**Step 1 — Install the plugin**
In Figma: Menu → Plugins → Browse all plugins → search "Google Sheets Sync" → Install. (Alternatively: "Data Populator" works the same way.)

**Step 2 — Understand the sheet structure**
When the `ad-copy-generation` workflow runs, it produces an assembled ad table with these column headers:
```
headline    body_copy    cta
```
These must match the Figma layer names exactly. They do, if you followed this spec.

**Step 3 — Connect the sheet to your master frame**
1. In Figma, select the master frame (e.g., "LinkedIn Single Image")
2. Run the plugin: Plugins → Google Sheets Sync → Connect
3. Paste the Google Sheet URL → authorize
4. Plugin shows a preview of row 1 populated into the frame
5. Click "Populate all rows" → plugin creates one duplicate frame per row

**Step 4 — Review**
Scroll through the generated frames. If any headline overflows, the text in the Sheet was too long (the workflow enforces 60 chars max, but double-check).

**Step 5 — Export**
Select all generated frames → File → Export → PNG 1× → Export.

**Step 6 — Upload to LinkedIn Ads / Google Ads**
Each exported frame is one ad creative. LinkedIn: upload under Images in the ad creation flow. Google: upload under Marketing images in Responsive Display Ads.

**What the plugin does NOT do:** Change colors, fonts, or the logo. Those are baked into the master frame. The plugin only swaps text content in the named layers.

## 6. Questions?

- **Hex values look off?** Check DevTools on aelira.in and update `core/brand/assets.md`.
- **Font isn't DM Sans?** Install the correct font via Google Fonts → update font in all text layers → update `core/brand/assets.md`.
- **Plugin not finding the sheet?** Make sure the Google Sheet is shared as "Anyone with the link can view" and the column headers are exactly: `headline`, `body_copy`, `cta` (no spaces, lowercase).
- **Need the white logo?** Ask the team to export it from the original brand file (AI, EPS, or Figma source). Upload to a CDN or Cloudinary and share the URL.
